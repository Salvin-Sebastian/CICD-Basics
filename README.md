# 🚀 CI/CD Pipeline Basics with GitHub Pages

## 📌 Project Overview
This project demonstrates how to implement a simple CI/CD (Continuous Integration and Continuous Deployment) pipeline using GitHub Actions and deploy a static website using GitHub Pages.

The pipeline automatically deploys the website whenever changes are pushed to the repository.

---

## 🎯 Objective
- Understand CI/CD concepts  
- Automate website deployment  
- Use GitHub Actions for workflow automation  
- Host a live website using GitHub Pages  

---

## 🛠️ Technologies Used
- HTML  
- GitHub  
- GitHub Actions  
- GitHub Pages  

---

## 📂 Project Structure
.
├── index.html
└── .github/
└── workflows/
└── deploy.yml

---

## ⚙️ CI/CD Workflow
The pipeline is configured to:

1. Trigger on every push to the `main` branch  
2. Checkout the repository code  
3. Deploy the website automatically to GitHub Pages
